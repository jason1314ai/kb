# KB Import — 信息录入 Skill

## 你的角色
将用户确认后的洞察/知识点，精准写入 `个人系统/[领域]/[子模块].md`。**增量追加，不删除已有内容。��入后更新内容索引。**

## KB 结构
```
Knowledge Base/
└── 个人系统/
    └── [领域]/
        ├── 内容索引.md     ← 每次新建子模块后必须更新
        └── [子模块].md     ← 写入目标
```

## 子模块文件模板

### 模板 A — 方法论型
新建方法论型子模块时使用：

```markdown
# [子模块名]

> 一句话：这个模块要解决什么问题

## 目标
我在这个领域想达到的状态是——

## 原则
- **原则名** — 展开解释

## SOP（可选）
1. 步骤一
2. 步骤二

## 方法论

### 场景：[场景名]
**适用：** 什么时候用
**做法：** 具体怎么做
```

### 模板 B — 知识资源型
新建知识资源型子模块时使用（概念解释、资源清单）：

```markdown
# [子模块名]

> 一句话定义这个概念/资源集合

## 是什么
核心定义或说明（2-5句）

## 核心要素
- **要素名** — 解释

## 关联概念
[[概念A]] · [[概念B]]

## 应用场景
- **场景** → 怎么用

## 资源清单（可选，清单型使用）
| 名称 | 类型 | 推荐理由 |
|------|------|----------|
| [书/工具] | 书/工具/课程 | |
```

## 内容索引模板
新建子模块后，在对应 `内容索引.md` 追加，并标注类型：

```markdown
### [子模块名].md `方法论`
> 这个子模块解决什么问题

- **目标** — [想达到的状态]
- **原则** — 关键词1 · 关键词2
- **方法论** — 场景A / 场景B
```

```markdown
### [子模块名].md `知识资源`
> 这个子模块覆盖什么概念/资源

- **核心要素** — 关键词1 · 关键词2
- **应用场景** — 场景A / 场景B
```

## 执行步骤

### Step 1：读取待写入内容
读取 kb-review 或 kb-article 输出的洞察列表，识别：
- 每条的归属路径（如 `个人系统/团队管理/向上管理.md`）
- 写入类型（追加原则 / 追加SOP步骤 / 新增场景 / 新建子模块）
- 用户���认的条目（跳过未确认的）

### Step 2：逐条执行写入

**方法论型 — 追加原则**
1. 读取目标子模块文件
2. 找到 `## 原则` 章节末尾
3. 追加新行：`- **[原则名]** — [内容]`

**方法论型 — 追加SOP步骤**
1. 读取目标子模块文件
2. 找到 `## SOP` 对应场景类型
3. 追加步骤；若无该场景类型，新建 `### [场景类型]` 小节

**方法论型 — 新增方法论场景**
1. 读取目标子模块文件
2. 在 `## 方法论` 末尾追加：
   ```
   ### 场景：[场景名]
   **适用：** [条件]
   **做法：** [内容]
   ```

**知识资源型 — 追加内容**
1. 读取目标子模块文件
2. 按内容类型找到对应章节（`## 是什么` / `## 核心要素` / `## 应用场景` / `## 资源清单`）末尾追加
3. 资源清单追加格式：`| [名称] | [类型] | [推荐理由] |`

**新建子模块（任意类型）**
1. 判断类型：方法论型用模板 A，知识资源型用模板 B
2. 在对应领域目录新建 `[子模块名].md`，填入内容
3. 在该领域 `内容索引.md` 追加对应类型的索引条目（含类型标注）

**新建领域（内容不属于任何现有领域时）**
1. 在 `个人系统/` 下新建文件夹 `[领域名]/`
2. 创建 `内容索引.md`，初始内容：
   ```markdown
   # [领域名] — 内容索引

   > 一句话定位这个领域的范围与目的

   ---
   ```
3. 再按「新建子模块」步骤写入内容
4. 告知用户已新建领域

### Step 3：写入后汇报并更新归档记录

```
## 写入完成

### 已写入
- 个人系统/团队管理/向上管理.md
  → 方法论/新增场景：「[场景名]」
- 个人系统/思维认知/决策框架.md
  → 原则/追加：「[原则名]」

### 新建文件
- 个人系统/职业发展/时间管理.md（新建子模块）
  → 内容索引已更新

### 已跳过
- 知识点3（用户跳过）
```

写入完成后，将本次来源文件追加到对应的归档列表：
- 来自 kb-article 的内容 → 追加到 `收纳箱/articles/archived.md` 的 `## 已归档` 列表
- 来自 kb-review 的内容 → 追加到 `收纳箱/reviews/archived.md` 的 `## 已归档` 列表
- 格式：`- \`[文件名]\``
- 若本次只导入了部分内容（用户跳过了某些条目），仍标记整个文件为已归档

## 核心约束
- ❌ 不删除任何已有内容
- ❌ 不修改用户已写的措辞
- ❌ 写入前不读取目标文件不动手
- ❌ 目标章节找不到时不强行插入（告知用户）
- ✅ 内容与现有内容重复时跳过并告知
- ✅ 每次新建子模块必须更新内容索引
