# KB Review — 复盘总结归档 Skill

## 你的角色
从复盘文件中提炼有价值的经验，识别归属领域和子模块，生成可直接写入个人系统的结构化洞察。最终通过 **kb-import** 执行写入。

## KB 结构
```
Knowledge Base/
├── 个人系统/
│   ├── 健康管理/[子模块].md
│   ├── 财务管理/[子模块].md
│   ├── 学习能力/[子模块].md
│   ├── 团队管理/[子模块].md
│   ├── 沟通推销/[子模块].md
│   ├── 技术领域/[子模块].md
│   ├── 思维认知/[子模块].md
│   └── 职业发展/[子模块].md
└── 收纳箱/reviews/        ← 本 skill 的输入
    └── YYYY-MM-DD-[类型]-[主题].md
```

## 执行步骤

### Step 1：确定时间范围
询问用户：**要处理哪个时间范围的复盘？**
- 示例：今天 / 本周 / 2026-04-01 至 2026-04-07 / 指定文件名 / 全部
- 读取 `收纳箱/reviews/archived.md` 中的已归档列表
- 扫描 `收纳箱/reviews/` 中符合范围的文件，排除 `archived.md`、`README.md` 及已归档列表中的文件
- 若无未处理文件，告知用户「该时间范围内暂无新复盘」
- 列出待处理文件后开始处理

### Step 2：解析复盘内容
复盘模板包含以下章节，逐一读取：
- **发生了什么（事实）**
- **做得好的地方**
- **需要改进的地方**
- **根本原因分析**
- **下一步行动**

### Step 3：提炼洞察，判断归属

对每条有价值的经验/教训：

**判断写入类型（复盘洞察通常为方法论型）：**
| 内容类型 | 写入位置 |
|----------|----------|
| 可复用的行为准则 | 子模块 `## 原则` |
| 可重复的操作步骤 | 子模块 `## SOP` |
| 特定情境的处理方式 | 子模块 `## 方法论 → 场景：[名称]` |

> 若复盘中包含概念解释或资源清单类内容，新建**知识资源型**子模块（见 kb-import 模板 B）。

**判断归属领域：**
健康管理 / 财务管理 / 学习能力 / 团队管理 / 沟通推销 / 技术领域 / 思维认知 / 职业发展
若内容不属于以上任何领域，建议新建领域（由 kb-import 创建文件夹和内容索引）。

**判断归属子模块：**
- 根据内容直接判断最匹配的子模块名，无需读取索引文件
- 子模块不存在时由 kb-import 负责新建

### Step 4：输出结构化洞察列表

以如下格式输出，供用户确认后交给 kb-import 执行：

```
## 复盘洞察 — [时间范围]
来源：[文件名列表]

---

### 洞察 1
**原文：** > [复盘原文摘录]
**归属：** 个人系统/团队管理/向上管理.md → ## 方法论
**写入类型：** 新增场景
**内容：**
#### 场景：[场景名]
**适用：** [什么情况下用]
**做法：**
1. [步骤]
2. [步骤]

---

### 洞察 2
**原文：** > [复盘原文摘录]
**归属：** 个人系统/思维认知/决策框架.md → ## 原则
**写入类型：** 追加原则
**内容：**
- **[原则名]** — [一句话说清楚]

---
```

完成后说：「已提炼 X 条洞察，告诉我哪些要导入，或说『全部导入』执行 kb-import。」

## 注意事项
- 「做得好的地方」如果可复用才提炼，不要把具体事件当原则
- 「根本原因分析」往往是最有价值的——优先提炼
- 同一个子模块同类型内容合并，不拆成多条
- 写入类型要准确：原则是通用准则，SOP是步骤，方法论/场景是情境应对
